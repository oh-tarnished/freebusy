## Services

### ChannelService

ChannelService is the distribution layer: it manages a property's connections to OTAs/GDS and drives 2-way ARI — pushing availability/rates/inventory out per mapped unit and landing reservations made on a channel back as bookings. The per-OTA adapters that actually talk to each channel run behind these methods.

### Channel

A connection between one property and one distribution channel (OTA/GDS). It is the anchor for 2-way ARI: availability/rates/inventory are pushed out per mapped unit, and reservations made on the channel are pulled in as bookings. Credentials are never carried in the API — only an opaque handle to where they are stored.
