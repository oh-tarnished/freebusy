# freebusy-client

Browser SDK for the FreeBusy gRPC-Web API. Built on [Connect-ES] and
[protobuf-es]: generated `*_pb.ts` message types + service descriptors, plus a
small factory that wires them onto a single [gRPC-Web transport].

## Install

```bash
bun install
```

Runtime deps: `@bufbuild/protobuf`, `@connectrpc/connect`,
`@connectrpc/connect-web`.

## Usage

```ts
import { createFreebusyClient, booking } from "freebusy-client";

const fb = createFreebusyClient({
  baseUrl: "https://api.freebusy.example",
  // any @connectrpc/connect-web GrpcWebTransportOptions is accepted, e.g.:
  // interceptors: [myAuthInterceptor],
  // fetch: customFetch,
});

// Connect-ES methods take a plain message object and return the response
// message directly (no { response } wrapper):
const res = await fb.availability.computeAvailability({
  resource: "resources/123",
  // ...request fields
});

// Message/enum types are grouped per service to avoid name clashes:
const req: booking.CreateBookingRequest = { /* ... */ };
```

### What's exported from `index.ts`

- `createFreebusyClient(options | transport)` — returns `{ availability, booking,
  identity, organisation, promoCode, resource, schedule, transport }`. Each is a
  Connect `Client<typeof XService>`.
- The service descriptors (`AvailabilityService`, `BookingService`, …) for
  building clients yourself with `createClient`.
- Connect primitives: `createClient`, `createGrpcWebTransport`, `ConnectError`,
  `Code`, `Interceptor`, `Transport`.
- Per-service type namespaces: `availability`, `booking`, `identity`,
  `organisation`, `promocode`, `resource`, `schedule`, `shared`.

## Layout

- `index.ts` — SDK entry point (factory + re-exports). Hand-written.
- `types/` — per-service barrels that group generated message types. Hand-written.
- `src/` — generated protobuf-es code (`*_pb.ts`). **Do not edit** — regenerated by buf.

## Regenerating

From the repo root:

```bash
just gen typescript   # buf generate --template tools/protobuf/buf/typescript.buf.gen.yaml
```

Generation targets `client/typescript/src` only. `clean: true` wipes that
directory on every run, so it must stay isolated from the hand-written
`index.ts`, `types/`, and `package.json` at the package root. Well-known types
(`google/protobuf/*`) are imported from `@bufbuild/protobuf/wkt` rather than
generated; the `google/api/*`, `google/type/*`, `mcp/*`, and `protorm/*` trees
are generated in-tree from the extra buf input modules.
